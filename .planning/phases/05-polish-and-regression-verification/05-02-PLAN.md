<objective>
Fix SVG flow line misalignment by computing path coordinates dynamically based on actual card width, and prevent circle/spacer compression at narrow widths.

Purpose: The hardcoded SVG path pixel offsets (e.g., M-100, h280, H160, H-120) assume 60px gaps between 80px circles (140px center-to-center). When the card renders narrower than rowMaxWidth (e.g., in the HA editor panel), flex space-between compresses gaps, but SVG paths don't adapt -- lines overshoot or undershoot target circles. This plan makes all SVG paths responsive to actual card width.
Output: Responsive SVG flow lines that align correctly at any card width.
</objective>

    **Part B: Dynamic SVG path coordinates (src/power-flow-card-plus.ts)**

    In the `render()` method, after the `const geo = computeFlowGeometry(...)` line (currently line 668) and the `--row-max-width` CSS property set, add a dynamic coordinate computation block:

    ```typescript
    // Dynamic SVG path coordinates: adapt to actual card width
    // At max width (rowMaxWidth), gap = 60px, reach = 100, lineLength = 280 (same as old hardcoded values)
    // At narrower widths, gap shrinks proportionally, so reach and lineLength adapt
    const actualRowWidth = Math.min(this._width || geo.rowMaxWidth, geo.rowMaxWidth);
    const gapBetweenCircles = geo.numCols > 1
      ? (actualRowWidth - geo.numCols * 80) / (geo.numCols - 1)
      : 0;
    const reach = gapBetweenCircles / 2 + 40; // distance from spacer edge to adjacent circle center
    // reach = 100 at max width (gap=60: 30+40=70... wait)
    ```

    IMPORTANT MATH CORRECTION: The spacer div is 80px wide. It sits between two circles. The SVG origin is at the spacer's top-left (0,0). To reach the CENTER of the LEFT adjacent circle:
    - Cross the gap between spacer left edge and circle right edge: `(gapBetweenCircles - 0) / 2`...

    Actually, let's think about this more carefully. With `space-between` layout:
    - Items are 80px wide each (circles and spacers are all 80px)
    - The gap between ANY two adjacent items = `(rowWidth - numItems * 80) / (numItems - 1)`

    But wait -- in the current layout, the "spacer" is one of the flex items. Each row has `numCols` items (mix of circle-containers and spacer divs), all 80px wide. The gap between items is the CSS space-between gap.

    The SVG is positioned at the spacer's top-left (0,0). To reach the center of the circle to the LEFT of the spacer:
    - Go left by: `gapBetweenItems / 2 + 40` (half the gap to reach the right edge of the circle, then 40px more to reach its center)

    But actually, the items in the row are: circle-container (80px), [gap], spacer (80px), [gap], circle-container (80px). The gap is the flex space-between gap. Since ALL items are 80px and there are `numCols` items:
    - Total gaps = numCols - 1
    - Each gap = (rowWidth - numCols * 80) / (numCols - 1)

    From the spacer's left edge (x=0) to the left circle's center:
    - x = -(gap + 40)  [go left across the gap, then 40px into the circle]

    But wait -- in the ACTUAL current hardcoded paths:
    - `M-100 40 h280` for a straight horizontal spanning one spacer
    - At max width: gap = 60, so leftReach = -(60 + 40) = -100. CHECK!
    - lineLength = 80 (spacer) + 60 (right gap) + 40 (into right circle) + 100 (left reach magnitude) = 280. CHECK!

    So the formula is:
    ```typescript
    const leftReach = -(gapBetweenCircles + 40);  // negative (going left from spacer origin)
    const rightReach = 80 + gapBetweenCircles + 40;  // going right past spacer + gap + into circle
    const straightLineLength = rightReach - leftReach;  // = 80 + 2*gap + 80 = 2*(gap+80)
    ```

    At max width: leftReach = -100, rightReach = 180, straightLineLength = 280. Matches hardcoded values.

      NOTE: The -120 vs -100 difference (20px extra) is intentional -- the solar-to-grid line ends deeper into the grid circle to visually separate from the grid-to-home line. Preserve this 20px offset: `leftReach - 20`.
